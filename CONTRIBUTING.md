# Contributing to HelixStudio AI

Thank you for your interest in contributing! We want to make contributing to HelixStudio as easy and transparent as possible.

## Our Development Process
We use GitHub issues to track public bugs and feature requests. Please ensure your PRs are descriptive and include tests for new functionality.

## Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.

## Code of Conduct
We are committed to providing a friendly, safe and welcoming environment for all, regardless of level of experience, gender identity and expression, sexual orientation, disability, personal appearance, body size, race, ethnicity, age, religion, or nationality.

## Security
